### Inactive Pane Transparency

Ghostty has a dedicated config option for this: **`unfocused-split-opacity`**.

The opacity level of an unfocused split controls how faded out inactive splits appear. Unfocused splits by default are slightly faded out to make it easier to see which split is focused. To disable this feature, set the value to 1.

A value of 1 is fully opaque and 0 is fully transparent. Because "0" is not useful, the minimum value is 0.15 — which "still looks weird but you can at least see what's going on." Values outside 0.15–1 are clamped.

Additionally, there's a companion option **`unfocused-split-fill`**:
This sets the color used to dim the unfocused split. Unfocused splits are dimmed by rendering a semi-transparent rectangle over the split. This defaults to the background color and can be specified as hex (`#RRGGBB`) or a named X11 color.

Example config:
```
unfocused-split-opacity = 0.7
unfocused-split-fill = #1a1a2e
```

### Other Major Customization Categories

Here's a broader overview of what Ghostty lets you configure:

### Appearance & Transparency
- **`background-opacity`** — the opacity of the background, from 0 (fully transparent) to 1 (fully opaque), with values clamped to valid range. On macOS, background opacity is disabled when the terminal enters native fullscreen.
- **`background-blur`** — blur effect behind the transparent background, supported on macOS and some Linux desktop environments. On macOS there are also glass effects (`macos-glass-regular`, `macos-glass-clear`).
- **`cursor-opacity`** — the opacity of the cursor, from 0 (fully transparent) to 1 (fully opaque).

### Themes
- Ghostty ships with hundreds of built-in themes that can be selected with one line of configuration.
- The built-in themes are sourced from iterm2-color-schemes and updated weekly.
- You can also author custom themes and set separate themes for light/dark mode.

### Fonts & Typography
- `font-family`, `font-size`, `font-thicken`, `font-feature` (for ligatures), `adjust-cell-height`
- Ghostty embeds a default font (JetBrains Mono) and has built-in nerd fonts.

### Keybindings
- Ghostty supports flexible, custom keybindings through the `keybind` configuration option.

### Splits & Panes
- Split vertically/horizontally, navigate between splits
- The unfocused split opacity/fill options discussed above

### Advanced Features
- **Custom shaders** — apply visual effects with GLSL shaders via `custom-shader` and `custom-shader-animation` (CRT scanlines, retro glow, etc.)
- **Quick terminal** — configure a quake-style dropdown terminal with `quick-terminal-position`, `quick-terminal-animation-duration`, etc.
- **Window options** — `window-decoration`, `window-height`, `window-width`, `macos-titlebar-style`, etc.
- **Minimum contrast ratio** — a configurable contrast ratio between foreground and background colors, valued between 1 and 21.

### Configuration Basics
Ghostty uses a simple `key = value` syntax in `~/.config/ghostty/config`. It supports hundreds of configuration options. You can also split configs across multiple files using `config-file` directives. To see all options with docs, run:

```bash
ghostty +show-config --default --docs
```

### Caveats
- On macOS, changing `background-opacity` requires restarting Ghostty completely (hot-reload doesn't apply it).
- `background-opacity` normally only applies to the window background. Cells with explicit background colors (e.g. from Neovim or Tmux) remain fully opaque, unless `background-opacity-cell` is used.

### 🎨 Visual Effects & Eye Candy

| Option | What it does |
|---|---|
| `custom-shader` | Custom shaders now receive terminal cursor information, enabling effects like a "cursor blaze" trail behind the cursor as it moves. You can stack multiple GLSL shaders (CRT scanlines, bloom, retro glow, etc.). Community repos like [ghostty-shaders](https://github.com/0xhckr/ghostty-shaders) and [cursor shaders](https://github.com/sahaj-b/ghostty-cursor-shaders) have ready-made effects. |
| `custom-shader-animation` | Controls whether shaders animate continuously. Set to `always` to animate even when unfocused (more CPU), `true` for focused-only, or `false` for static. |
| `background-image` | Set a PNG/JPEG as a terminal background image — with sub-options for `background-image-opacity`, `background-image-position` (`center`, `top-left`, etc.), `background-image-fit` (`contain`, `cover`, `stretch`, `none`), and `background-image-repeat`. *(Since 1.2.0)* |
| `background-blur` | On macOS 26.0+, special values `macos-glass-regular` and `macos-glass-clear` enable native glass effects. On Linux KDE, it integrates with KWin's global blur. |
| `window-colorspace` | Switch between `srgb` and `display-p3` for interpreting terminal colors. macOS only, but notable for color-accurate work. |

### ⌨️ Keybinding Power Features

| Option | What it does |
|---|---|
| `performable:` prefix | A keybinding is activated only if the action can actually be performed. E.g., `performable:ctrl+c=copy_to_clipboard` copies only when there's a selection — otherwise it sends the interrupt signal. |
| Key sequences (chords) | Bind multi-key combos like `ctrl+a>n=new_window` — a leader-key system with no timeout. |
| Key Tables *(1.3.0)* | Create named modal keybinding sets (like vim modes). Activate/deactivate with `activate_key_table:<name>`. You can build a full "copy mode" or "navigation mode" this way. |
| Chained Actions *(1.3.0)* | A single keybind can trigger multiple sequential actions using `chain=`. |
| `key-remap` | Remap modifier keys at the app level — e.g., `key-remap = ctrl=super` to swap Ctrl and Cmd within Ghostty only. |
| `global:` prefix | Make a keybind system-wide — works even when Ghostty isn't focused. Supported on macOS and GNOME 48+/KDE Plasma 5.27+. |

### 🖥️ Quick Terminal (Quake-style dropdown)

| Option | What it does |
|---|---|
| `quick-terminal-position` | `top`, `bottom`, `left`, `right`, or `center`. |
| `quick-terminal-size` | Percentage or pixel-based, with separate primary/secondary axis sizes (e.g. `50%,500px`). |
| `quick-terminal-animation-duration` | Control the slide-in/out speed (set to `0` to disable animation). |
| `quick-terminal-autohide` | Auto-hide when focus leaves (default `true` on macOS, `false` on Linux). |
| `gtk-quick-terminal-layer` | On Wayland, control z-ordering: `overlay`, `top`, `bottom`, or `background`. |

### 🔧 Behavioral / QoL Gems

| Option | What it does |
|---|---|
| `cursor-click-to-move` | Click anywhere in the prompt to reposition the cursor — requires shell integration (works natively with Fish v4, Nu 0.111+). |
| `notify-on-command-finish` *(1.3.0)* | Get notified when a long-running command finishes. Configurable threshold (`notify-on-command-finish-after = 30s`) and action (`bell`, `notify`). |
| `minimum-contrast` | The contrast ratio between foreground and background, valued 1–21 per WCAG 2.0. A value of 1.1 avoids invisible text; 3+ avoids hard-to-read text. Great accessibility feature. |
| `window-save-state` | On macOS, set to `always` to persist window positions, tabs, splits, and working directories across restarts. |
| `focus-follows-mouse` | In split-pane mode, hovering over a split auto-focuses it — no click needed. |
| `mouse-reporting = false` | Completely disable mouse reporting to applications. Useful if you always want native selection behavior. |
| `clipboard-paste-protection` | Warns before pasting text that looks dangerous (contains newlines, potential command injection). |
| `wait-after-command` | Keep the terminal open after the command exits — great for scripts and debugging. |
| `input` | Pre-send data to the terminal on startup: `input = "neofetch\n"` will run a command immediately. Supports raw strings and file paths. |
| `selection-word-chars` *(1.3.0)* | Customize word boundary characters for double-click selection. |

### 🍎 macOS-Specific Highlights

| Option | What it does |
|---|---|
| `macos-icon` | Choose from built-in icon variants: `blueprint`, `holographic`, `retro`, `xray`, `glass`, etc. Or use `custom-style` to set your own `macos-icon-ghost-color` and `macos-icon-screen-color` (gradient!). |
| `macos-titlebar-style` | `tabs` merges the tab bar into the titlebar for a clean look. `transparent` lets your background color bleed through. `hidden` removes the titlebar but keeps rounded corners. |

### 🐧 Linux-Specific Highlights

| Option | What it does |
|---|---|
| `linux-cgroup` | Put every surface into a transient systemd scope for per-tab/split resource isolation. If one tab OOMs, only that tab gets killed — not all of Ghostty. |
| `linux-cgroup-memory-limit` | Set a soft memory limit per terminal surface. |
| `gtk-custom-css` | Load arbitrary GTK4 CSS files to restyle Ghostty's chrome. Use `GTK_DEBUG=interactive ghostty` to live-edit. |

### 🔤 Typography Deep Cuts

| Option | What it does |
|---|---|
| `font-codepoint-map` | Force specific Unicode ranges to use a specific font — e.g., map all CJK characters to Noto Sans CJK while keeping your main font for Latin. |
| `font-variation` | Set variable font axes directly: `font-variation = wght=300` for light weight, `slnt=-12` for slant, etc. |
| `font-shaping-break` *(1.2.0)* | Control where ligatures break — by default, ligatures break under the cursor so you can see individual characters while editing. |
| `palette-generate` *(1.3.0)* | Auto-generate the 256-color palette from just the 16 ANSI colors — gives a cohesive look without specifying all 256 manually. |
| `bold-color` | Set a specific color for bold text, or use `bright` to map bold to bright palette colors. |
