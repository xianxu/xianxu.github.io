**GLSL** stands for **OpenGL Shading Language**. It's a C-like programming language used to write small programs called **shaders** that run directly on your **GPU** (graphics card).

### What are shaders?

Shaders are tiny programs that tell the GPU how to render each pixel (or vertex) on screen. They run massively in parallel — one instance per pixel — which makes them extremely fast for visual effects.

There are a few types, but the one relevant to Ghostty is the **fragment shader** (also called a pixel shader): it determines the final color of each pixel on screen.

### How does this relate to Ghostty?

Ghostty renders its terminal using the GPU (via OpenGL/Metal). It allows you to inject your own custom fragment shaders that post-process the terminal output. This means you can apply visual effects like:

- **CRT screen simulation** (scanlines, curvature, bloom)
- **Color grading / tinting**
- **Blur or distortion effects**
- **Animated backgrounds**
- **Vignette, film grain, etc.**

### A tiny example

A simple GLSL fragment shader might look like this:

```glsl
// Inverts the colors of the terminal
vec4 color = texture(iChannel0, uv);
fragColor = vec4(1.0 - color.rgb, color.a);
```

Each frame, this runs for *every pixel*, reads the terminal's rendered color, and inverts it.

### In short

| Term | Meaning |
|------|---------|
| **GL** | OpenGL — a cross-platform graphics API |
| **SL** | Shading Language |
| **Shader** | A small GPU program that controls how pixels are drawn |
| **Fragment shader** | Specifically controls the *color of each pixel* |

Think of it as a programmable Instagram filter, but running on your GPU in real-time, applied to your terminal output. It's powerful but niche — most people won't need it, but it's a fun way to deeply customize Ghostty's appearance.
