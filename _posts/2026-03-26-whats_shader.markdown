Ghostty's **custom shader** feature lets you apply GPU fragment shaders (written in GLSL) as post-processing effects on the terminal's rendered output. It's one of the more unique features among terminal emulators.

### How It Works

Ghostty is a GPU-powered terminal emulator. It uses raw Metal on macOS and OpenGL on Linux. GPUs render things by invoking shaders — a shader is essentially a program that runs on the GPU. Shaders are generally written in a technology-specific programming language.

Ghostty accepts shaders written in GLSL and will convert them on the fly to MSL on macOS. To do this, Ghostty uses **glslang** to convert GLSL to **SPIR-V** (an intermediate representation) and then **SPIRV-Cross** to convert SPIR-V to the target format such as MSL.

### Shadertoy Compatibility

This is the killer detail: Ghostty exposes the **Shadertoy interface**, so you can take many Shadertoy shaders and drop them into Ghostty without any modification. More importantly, you can use Shadertoy itself as a live, interactive development tool for Ghostty custom shaders.

This means the massive library of effects on [shadertoy.com](https://www.shadertoy.com/) is a starting point — CRT effects, noise, water, fractals, etc.

### Configuration

It's simple — just point to `.glsl` files in your config:

```ini
# ~/.config/ghostty/config
custom-shader = ~/.config/ghostty/shaders/crt.glsl
custom-shader-animation = true   # or "always" to animate even unfocused windows
```

You can specify multiple shaders and they will be applied in order (chained). For example, stacking a TFT shader + CRT + bloom:

```ini
custom-shader = ./shaders/tft.glsl
custom-shader = ./shaders/bettercrt.glsl
custom-shader = ./shaders/bloom.glsl
```

As of Ghostty 1.2, custom shaders can also be hot reloaded.

### Cursor Uniforms (1.2+)

Custom shaders are now provided information about the terminal cursor, so that custom effects and animations can be applied to it, like a trail or smear. This enables things like "cursor blaze" trails that follow the cursor as it moves. Note: the cursor trail shader can only show trails for the "hardware" cursor. Helix, because of its multiple cursor support, often hides the hardware cursor and shows a "fake" cursor using various techniques. The cursor shader knows nothing about those.

### Practical Uses

Beyond fun/aesthetics, the primary practical use case for custom shaders is **accessibility**. Custom shaders are a great way to specifically address certain forms of colorblindness, affect contrast or brightness to make things more readable, create "magnifying glass" effects, etc.

### Performance

On Mitchell's machine, CRT-style effects use ~1% CPU and ~2% GPU. The feature does not negatively impact performance. Ghostty uses a multi-threaded rendering architecture and has plenty of idle time between frames. When custom shaders are in use, there is less idle time but it has very little to no effect on the other threads (IO and GUI). When custom shaders are not in use, Ghostty uses the same rendering pipeline it always has.

### Community Shader Repos

There's a growing ecosystem of ready-to-use shaders:

- [ghostty-shaders by 0xhckr](https://github.com/0xhckr/ghostty-shaders) — large collection (CRT, noise, water, gears, gradients, etc.)
- [ghostty-cursor-shaders by sahaj-b](https://github.com/sahaj-b/ghostty-cursor-shaders) — cursor trails and ripple/pulse effects with customizable parameters
- [CRT shaders by thijskok](https://github.com/thijskok/ghostty-shaders) — amber, green, and blue phosphor CRT looks
- [Watercolor shaders](http://jrmeyer.github.io/misc/2026/02/21/ghostty-watercolors.html) — randomized watercolor wash backgrounds

### Gotcha

Some shaders may appear to not work because the default background color covers them up. If you're using background-modifying shaders, you may need to explicitly set `background = #000000` or use `background-opacity` to let the shader show through.
