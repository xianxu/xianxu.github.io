In Ghostty (and many terminal emulators), "splits" refer to splitting the terminal window into multiple panes — similar to tmux or iTerm2 splits. "Navigate between splits" means moving focus from one split pane to another. Let me explain this concisely since the user seems to be exploring Ghostty's features.

In Ghostty, splits are multiple terminal panes within a single window — you can divide your window horizontally or vertically, each running its own shell session (similar to tmux panes or iTerm2 splits).

"Navigate between splits" simply means moving your keyboard focus from one pane to another. Ghostty provides keybindings for this:

Default Keybindings (macOS)

| Action | Default Keybinding | |---|---| | Split right (vertical) | Cmd + D | | Split down (horizontal) | Cmd + Shift + D | | Navigate to split above | Ctrl + Alt + ↑ (or Cmd + Alt + ↑ on macOS) | | Navigate to split below | Ctrl + Alt + ↓ | | Navigate to split left | Ctrl + Alt + ← | | Navigate to split right | Ctrl + Alt + → | | Close split | Cmd + W |

Why It Matters

Without split navigation keybindings, you'd have to click with your mouse to focus a different pane. The keybindings let you stay fully keyboard-driven.

This is also where unfocused-split-opacity becomes relevant — it visually dims the panes you're not focused on, making it obvious which split is active as you navigate between them.

You can customize all of these keybindings in your Ghostty config file using the keybind directive, e.g.: 

keybind = ctrl+h=goto_split:left
keybind = ctrl+j=goto_split:bottom
keybind = ctrl+k=goto_split:top
keybind = ctrl+l=goto_split:right

These vim-style bindings (h/j/k/l) are a popular customization.
